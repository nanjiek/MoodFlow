# MoodFlow 用户视角优化计划

## 目标

让用户更容易记录情绪、更愿意持续使用，并且觉得分析结果“准、暖、有帮助”，而不是一个冷冰冰的情绪分类工具。

## P0：先优化核心体验

### 1. 降低情绪记录门槛

- 首页直接进入“记录今天情绪”
- 支持 30 秒内完成：选情绪、写一句话、选表情
- 文本可选填，避免用户没话写时放弃
- 提供引导句，例如“今天让你印象最深的一件事是什么？”

### 2. 优化情绪标签

- 当前 8 类情绪可以保留，但用户端展示要更生活化
- 例如：
  - 焦虑：有点慌
  - 难过：心里低落
  - 疲惫：没电了
  - 平淡：没什么波动
- 避免用户觉得标签太专业、太像问卷

### 3. 分析结果要可解释

- 不只显示“你是焦虑”
- 应显示：
  - 当前情绪
  - 可能关键词
  - 为什么这样判断
  - 一句温和反馈
- 示例：

> 你今天多次提到“压力、担心、汇报”，系统判断你可能处在焦虑状态。

### 4. 陪伴内容要更贴合

- 不同情绪对应不同建议
- 焦虑：呼吸练习、任务拆分
- 难过：安慰话术、情绪日记
- 疲惫：休息提醒、低刺激音乐
- 开心/期待：记录积极事件，强化正反馈

## P1：提升长期使用感

### 5. 成长曲线不要只做图表

- 用户不一定关心复杂统计
- 应转化为易懂总结：
  - “这周你有 3 天比较焦虑”
  - “周三之后情绪明显变平稳”
  - “疲惫常出现在晚上记录中”
- 图表作为辅助，不作为唯一反馈

### 6. 增加轻量复盘

- 每周自动生成一份情绪小结
- 包含：
  - 本周主要情绪
  - 高频关键词
  - 情绪变化
  - 一个下周建议
- 语气要像陪伴，而不是诊断

### 7. 提醒机制要克制

- 默认一天最多一次提醒
- 允许用户关闭
- 连续负面情绪时提醒要温和：
  - “最近你好像有点累，要不要休息一下？”
- 避免制造压力感

## P2：增强信任与安全感

### 8. 隐私感知要明显

- 明确告诉用户：
  - 情绪记录仅自己可见
  - 树洞是匿名的
  - 可删除历史记录
- 对敏感内容提供“加密记录”入口

### 9. 避免医疗化表达

- 不使用“诊断、病症、治疗”等表达
- 使用“可能、倾向、建议、陪你看看”这类语气
- 防止用户误以为产品在做心理疾病判断

### 10. 模型结果允许用户纠正

- 如果系统识别错了，用户可以点“不是这个情绪”
- 纠正结果进入训练样本
- 这能提升体验，也能改善模型数据

## P3：数据与模型体验优化

### 11. 重点补充不平衡情绪数据

- 当前 `疲惫 tired` 数据严重不足
- 用户端如果频繁识别错“疲惫”，体验会很差
- 优先补充：
  - 疲惫
  - 期待
  - 平静
  - 难过

### 12. 建立真实用户反馈闭环

- 记录：
  - 用户是否接受预测结果
  - 是否点击陪伴内容
  - 是否完成调节工具
- 用这些数据优化推荐和模型

## 阶段计划

```text
第一阶段：优化记录流程、标签文案、分析结果展示
第二阶段：优化陪伴内容、成长曲线、每周复盘
第三阶段：补充隐私保护、用户纠错、数据反馈闭环
第四阶段：基于真实数据重新训练模型，提升识别准确率
```

## 一句话重点

优先把 MoodFlow 做成“用户愿意每天打开、记录后能被理解”的产品，再追求复杂算法和高级功能。
