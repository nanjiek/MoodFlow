# MoodFlow 公开仓库前脱敏清单

更新时间：2026-05-17

这份清单用于说明：当前仓库中哪些内容已经适合公开，哪些内容是为了继续本地联调和 APK 打包而**暂时保留**的环境绑定信息，以及在正式公开 / 回传仓库 / 提交 PR 前建议再做哪些处理。

## 1. 当前已经清理完成的内容

以下内容目前已经基本适合直接公开：

- `README.md`
- `docs/expo_debug_setup.md`
- `docs/android_packaging_checklist.md`
- `mobile/README.md`
- `mobile/.env.example`

当前这些文档已经完成了以下清理：

- 去掉了本机绝对路径，如 `C:\Users\...`
- 去掉了直接写死的本机局域网地址说明
- 将说明改成了占位形式，如：
  - `http://<your-lan-ip>:8000`
  - `http://<your-lan-ip>:8010`

## 2. 当前仍然保留的环境绑定信息

下面这些内容**不是文档泄露问题**，但它们确实仍然绑定着当前这台机器或当前 Expo / EAS 项目：

### 2.1 `mobile/eas.json`

当前保留了固定的构建环境变量：

- `EXPO_PUBLIC_API_BASE_URL = http://192.168.1.5:8000`
- `EXPO_PUBLIC_MODEL_BASE_URL = http://192.168.1.5:8010`

保留原因：

- 这是当前 APK 打包能够直接连到本机后端的关键配置
- 如果现在改成占位值，会破坏已经跑通的 EAS 打包链路

影响：

- 其他人在自己的电脑上打包前，需要先把这里改成自己的局域网 IP

### 2.2 `mobile/app.json`

当前仍保留：

- `owner: "niujl"`
- `extra.eas.projectId: "693a5f09-4680-46ac-b083-c202f9464cee"`

保留原因：

- 这是当前 Expo / EAS 项目绑定信息
- 继续使用现有打包链时需要这些配置

影响：

- 如果后续换 Expo 账号、换 EAS 项目，或准备把仓库完全交给别人维护，需要重新绑定

## 3. 当前建议：先保留，不改源码

基于当前阶段的目标，建议如下：

- **继续保留** `mobile/eas.json` 中的当前局域网 IP
- **继续保留** `mobile/app.json` 中的 `owner` 和 `projectId`

原因：

- 当前项目还在继续联调和真机 APK 测试
- 这些配置一旦提前清空，会直接影响后续打包与复测

也就是说，当前阶段更适合：

- 在文档中明确说明这些值需要按接手者环境修改
- 但暂时不在源码里把它们去掉

## 4. 如果后续要公开仓库，建议再做的处理

当满足以下任一条件时，建议再做一次“公开前脱敏”：

- 准备将仓库完全公开到 GitHub
- 准备让其他同学 / 审核者自行接手打包
- 准备从当前 Expo / EAS 项目迁移到新的账号或组织

建议处理项如下：

### 4.1 `mobile/eas.json`

可改成以下两种方式之一：

1. 保留字段结构，但改成占位说明值
2. 删除具体 IP，并在文档里说明构建前必须自行填写

### 4.2 `mobile/app.json`

根据后续目标选择：

- 如果继续使用当前 Expo / EAS 项目：保留
- 如果准备完全公开模板化：移除 `owner` 和 `projectId`，由接手者重新执行 EAS 绑定

### 4.3 Firebase 相关文件

当前仓库中有：

- `mobile/google-services.json`

公开前建议再次确认：

- 是否允许作为当前项目配置继续保留
- 是否需要换成接手者自己的 Firebase 项目配置

说明：

- `google-services.json` 本身不是私钥
- 但它绑定了当前 Firebase 项目

### 4.4 Firebase Admin Service Account 私钥

这类文件**不应进入仓库**。

当前约定是：

- Firebase Admin 私钥仅保存在本机安全路径
- 通过 EAS 上传，不直接提交到仓库

公开前应再次确认：

- 仓库中没有任何 Firebase Admin 私钥 JSON
- 文档中不包含该文件内容

当前推荐分两阶段处理：

### 阶段一：继续开发 / 联调 / 打包

- 保留现有 `app.json` / `eas.json`
- 通过 `README.md` 告知接手者如何改成自己的 IP

- 再做一次“公开版脱敏”
- 视需要去掉：
  - `owner`
  - `projectId`
  - 固定局域网 IP

当前仓库的**文档层面已经基本清理干净**；仍然保留的主要是为了维持现有 Expo / EAS / APK 测试链路所必需的环境绑定配置。  
如果后续要进入完全公开或交接阶段，再对 `mobile/app.json` 和 `mobile/eas.json` 做第二轮模板化处理会更稳。
